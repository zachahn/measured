- Always opt to use the standard library. Never install any libraries.
- Never use an em-dash. Never use a spaced hyphen as a stand-in. An em-dash tacks an extra clause onto a finished sentence and signals that you are choosing to keep talking past the point. End the sentence at the period. If the trailing clause matters, make it its own sentence. If it does not, cut it.

## Writing skills and agents as SOPs

Treat every skill and agent as a Standard Operating Procedure (SOP): a document that walks an LLM through one task. The reader wants to finish the job and go home, makes poor decisions when left to choose, and works from a task list. Write for that reader.

- **Informative, not exhaustive.** Give only what the task needs. Cut background the LLM will skim past.
- **Simple.** Remove choices. State the one right way to do each step instead of listing options and trade-offs.
- **Step by step.** Lay the task out as an ordered list of concrete actions. Break long steps into sub-steps so each one is a single, clear action.
- **Self-contained.** Write so the LLM can complete the task from the SOP alone, without hunting for context.
- **Keep it current.** Update the SOP when the process, tools, or commands change.

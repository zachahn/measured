Rewrite it and hold to these rules:

- Write every sentence as subject, verb, object.
- Name the thing that acts. Write "the parser", "the approval screen", "the retry loop". Never write "the mechanism", "the component", "the form".
- Turn nominalizations back into verbs. Write "the compiler rejects the file, so the deploy stops". Never write "compilation failure results in deployment cessation".
- Never stack abstract nouns into a slogan. Write "The engine runs both skills and agents through one code path". Never write "one runtime mechanism, two authoring forms".
- Write active voice. Name the actor, then the verb. Write "The engine logs every call". Never write "every call is logged by the engine".
- Write one idea per sentence. Split a sentence that needs three commas into short full sentences.
- Make every pronoun point at one obvious noun. Do not coin jargon the reader cannot picture. State a point once. Do not restate it as a tagline.
- Use simple words. Write "use", not "utilize". Omit adverbs. Put statements in positive form. Do not overstate.
</guidance>
